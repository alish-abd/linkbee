# LinkBee - Платформа для создания био-страниц с ИИ

Современная платформа для создания био-страниц (link in bio) с помощью искусственного интеллекта. Пользователи описывают свою деятельность, добавляют ссылки и изображения, а ИИ создает красивую страницу за несколько секунд.

## Технологии

- **Next.js 14** - React фреймворк с App Router
- **TypeScript** - типизация
- **Tailwind CSS** - стилизация
- **Supabase** - бэкенд (база данных, аутентификация, storage)
- **OpenAI GPT-4** - генерация контента с помощью ИИ
- **React 18** - UI библиотека

## Функциональность

- ✅ Регистрация и аутентификация пользователей
- ✅ Создание био-страниц с помощью ИИ
- ✅ Загрузка изображений
- ✅ Управление ссылками
- ✅ Выбор темы и настройка дизайна
- ✅ Публичные страницы с уникальными URL
- ✅ Редактирование страниц
- ✅ Личный кабинет

## Установка

1. Клонируйте репозиторий и установите зависимости:
```bash
npm install
```

2. Создайте проект в Supabase:
   - Перейдите на [supabase.com](https://supabase.com)
   - Создайте новый проект
   - Запишите URL проекта и API ключи

3. ✅ Миграции базы данных применены автоматически:
   - Таблицы созданы: `users`, `bio_pages`, `links`, `images`
   - RLS политики настроены
   - Storage bucket `bio-images` создан
   - Политики Storage настроены
   
   Все миграции уже применены к вашему проекту Supabase!

5. Настройте переменные окружения:
   - Скопируйте `.env.local.example` в `.env.local`
   - Заполните переменные:
     ```env
     NEXT_PUBLIC_SUPABASE_URL=your-supabase-project-url
     NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
     SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key
     OPENAI_API_KEY=your-openai-api-key
     ```

6. Запустите dev сервер:
```bash
npm run dev
```

7. Откройте [http://localhost:3000](http://localhost:3000) в браузере

## База данных

### Таблицы

- **users** - профили пользователей
- **bio_pages** - био-страницы
- **links** - ссылки на страницах
- **images** - изображения

### Row Level Security (RLS)

Все таблицы защищены RLS политиками:
- Пользователи могут управлять только своими данными
- Публичные страницы доступны всем (read-only)
- Storage: публичный доступ на чтение, запись только для авторизованных

## API Routes

- `POST /api/upload` - Загрузка изображений
- `POST /api/generate` - Генерация страницы с ИИ
- `GET /api/bio-pages` - Список био-страниц пользователя
- `POST /api/bio-pages` - Создание био-страницы
- `GET /api/bio-pages/[id]` - Получение био-страницы
- `PUT /api/bio-pages/[id]` - Обновление био-страницы
- `DELETE /api/bio-pages/[id]` - Удаление био-страницы
- `PUT /api/users/profile` - Обновление профиля

## Генерация с ИИ

Платформа использует OpenAI GPT-4 для генерации контента:
- Структура страницы (блоки, порядок)
- Тексты для блоков
- Цветовая схема и стиль
- Рекомендации по размещению изображений
